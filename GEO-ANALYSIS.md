## 4. llms.txt Status

**Present:** Yes, at https://www.degrotemarketing.nl/llms.txt
**Generated by:** Yoast SEO v27.6
**Format:** Valid llms.txt structure (# heading, ## sections, markdown links)

### What is present
- Site title heading
- Pages section (homepage only)
- Categories section (3 categories: jij, Uncategorized, wij)
- Tags section (3 tags: SEO, Groningen, lokale SEO)
- Optional section (sitemap index link)

### What is missing

1. **RSL 1.0 license declaration.** Memory notes "RSL 1.0 implemented" but the live file contains no license header. RSL 1.0 (Robots Scraping License) requires a line like `License: https://rsl.ai/1.0` near the top of the file. This is not present. Either the implementation was not deployed or Yoast stripped it on regeneration.

2. **Service pages not listed.** The four service pages (/seo-groningen/, /google-ads-groningen/, /contentmarketing-groningen/, /website-groningen/) are not referenced in llms.txt. An AI model reading this file gets no signal that these pages exist.

3. **Blog posts not listed.** Nine published blog posts are invisible in llms.txt. Only the homepage appears under Pages.

4. **Category names are uninformative.** "jij", "wij", and "Uncategorized" give no semantic signal to an LLM about content topic areas.

5. **No description field.** The llms.txt spec allows a description block after the title. Currently omitted.

**Verdict: File exists and is syntactically valid, but is content-poor. Yoast's default output is minimal. Needs a handwritten or supplemented llms.txt.**

## 8. Top 5 Highest-Impact Changes

Ordered by expected GEO score impact, not effort.

### Change 1: Add FAQPage schema to all service pages
**Estimated score impact: +6 to +8 points (Citability + Structural Readability)**
**Effort: Medium (2-4 hours)**

Google explicitly supports FAQPage structured data in AI Overviews. Each service page should have 3-5 Q&A pairs answering real intent questions:

Example for SEO Groningen:
- "Wat kost SEO in Groningen?" - answer with price range and factors
- "Hoe lang duurt het voor ik hoger sta in Google?" - answer with honest timeline
- "Wat is het verschil tussen SEO en Google Ads?" - answer in 60-80 words

In functions.php, extend `dgm_schema()` for service page templates to output a `FAQPage` node. Questions should mirror the FAQ content written into the page HTML so content and schema match.

This change directly increases the probability of being surfaced in Google AI Overviews for local service queries. Google's official documentation confirms FAQPage rich results can appear in AI-generated responses.

---

### Change 2: Rewrite service page opening sections as direct-answer passages
**Estimated score impact: +5 to +7 points (Citability)**
**Effort: Medium (3-5 hours)**

Every service page currently opens with a brand-voice statement ("Gewoon doen", "Wij maken dat makkelijk"). These are not citable by AI systems because they do not answer a question.

Replace the first body paragraph of each service page with a 134-167 word self-contained answer to the primary search intent. The brand voice can follow in the second paragraph.

Example rewrite for SEO Groningen:
Current: "Wij maken dat makkelijk. Geen abracadabra. Gewoon gevonden worden."
Better: "SEO voor Groningse ondernemers betekent drie dingen: je website vertelt Google duidelijk wat je doet en waar je zit, elke dienst heeft een eigen pagina die aansluit op een echte zoekvraag, en je bouwt maandelijks aan autoriteit via vermeldingen en links. Een lokaal bedrijf in Groningen dat dit consistent doet gedurende zes maanden staat in de meeste gevallen boven 80% van de directe concurrentie in de zoekresultaten. De tijdsinvestering is realistisch: een uur per week volstaat voor de basis. De eerste resultaten zijn zichtbaar na 6-12 weken."

This passage is self-contained, 134 words, answers "wat is SEO Groningen" directly, and contains a specific claim.

- GeoCoordinates

### What to add

**Priority 1: FAQPage on service pages**
See Change 1 above. This is the highest-value schema addition for AI Overviews visibility.

**Priority 2: HowTo schema for procedural blog posts**
The "SEO voor Grunneger ondernemers" post has three numbered steps. Add HowTo schema:
```json
{
  "@type": "HowTo",
  "name": "SEO voor Groningse ondernemers: 3 stappen",
  "step": [
    {"@type": "HowToStep", "name": "Zorg dat Google snapt wie je bent en waar je zit", "text": "..."},
    {"@type": "HowToStep", "name": "Schrijf een pagina per zoekvraag", "text": "..."},
    {"@type": "HowToStep", "name": "Een goede backlink per maand", "text": "..."}
  ]
}
```

**Priority 3: Review schema on service pages**
The organization schema has `aggregateRating` but it only appears in the global graph. Add individual Review nodes or link the existing AggregateRating to specific service pages so rich results can appear per service.

**Priority 4: Add `knowsAbout` to service page WebPage nodes**
Currently WebPage nodes only have `about` pointing to the organization. Adding `about` pointing to the specific service entity and `keywords` makes the page topic clearer to knowledge graph parsers.

**Priority 5: foundingDate accuracy**
Schema states `foundingDate: 2026-05-01`. If the business predates this date under any form, update to the actual founding date. Knowledge graph validators flag implausibly recent founding dates.

## 10. Content Reformatting Suggestions

These changes do not require new content - they reformat existing content for better AI extractability.

### 10.1 Convert prose lists to HTML lists

Several pages describe multi-step processes in running paragraphs. Example from Google Ads page (four-step process described in text). Convert to `<ol>` with `<li>` items. AI systems extract structured lists more reliably than embedded sequences.

### 10.2 Add definition-style callout blocks

Add a styled `<div>` or `<blockquote>` at the start of each H2 section that answers the section's implied question in 2-3 sentences. This creates an extractable "answer block." Trafilatura (used by many AI pipelines) strips navigation and pulls main content - a clearly bounded answer block survives this extraction better than embedded prose.

Example pattern:
```html
<div class="answer-block">
  <strong>Wat is lokale SEO?</strong>
  Lokale SEO zorgt ervoor dat jouw bedrijf verschijnt als mensen in jouw stad zoeken op jouw dienst.
  Voor Groningse ondernemers betekent dit: je Google Bedrijfsprofiel optimaliseren,
  lokale zoekwoorden gebruiken op je website, en vermeldingen verzamelen in lokale directories.
</div>
```

### 10.3 Rewrite H2 headings on service pages as questions

Current service page H2 examples:
- "GEVONDEN WORDEN OF NIET" - not a question
- "WAT WE DOEN" - not a question
- "MORGEN BOVENAAN IN GOOGLE" - not a question

Rewrite as:
- "Hoe kom ik hoger in Google in Groningen?"
- "Wat doet een SEO-bureau in Groningen?"
- "Hoe snel zie ik resultaat van Google Ads?"

Question-based headings are the single most reliable structural signal for AI Overviews to identify a passage as a candidate answer. This aligns with Google's guidance on "helpful content" - write for users asking questions.

### 10.4 Add a "Veelgestelde vragen" section to each service page

Add 3-5 Q&A pairs at the bottom of each service page in `<details>/<summary>` or plain `<h3>`/`<p>` format (either works for AI extraction; `<details>` may be ignored by some crawlers). Pair with FAQPage schema (Change 1). This creates the most reliable AI Overview trigger pattern.

### 10.5 Standardize blog post length

Current posts range from ~270 to ~650 words. The shortest posts (under 400 words) do not contain enough content for AI systems to establish topical authority. Target a minimum of 600 words per post, with at least one passage between 134-167 words that can stand alone as a quoted answer.
